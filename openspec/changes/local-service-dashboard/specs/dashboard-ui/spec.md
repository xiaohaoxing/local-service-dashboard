## ADDED Requirements

### Requirement: 服务列表展示
系统 SHALL 在主页以卡片网格或列表形式展示所有注册的服务，每张卡片显示：服务名称、URL、描述、标签、在线状态指示（绿点/红点）、图标（emoji 或首字母）。

#### Scenario: 展示服务卡片
- **WHEN** 用户访问导航站首页
- **THEN** 页面显示所有已注册服务的卡片列表，每张卡片包含名称、URL 链接、在线状态

#### Scenario: 点击服务卡片打开链接
- **WHEN** 用户点击某服务卡片
- **THEN** 浏览器在新标签页中打开该服务的 URL

#### Scenario: 无服务时显示空状态
- **WHEN** 注册表中没有任何服务
- **THEN** 页面显示空状态提示，引导用户手动添加或执行端口扫描

### Requirement: 服务搜索与过滤
系统 SHALL 提供搜索框和标签过滤，让用户快速找到目标服务。

#### Scenario: 按名称搜索
- **WHEN** 用户在搜索框输入关键词
- **THEN** 列表实时过滤，仅显示名称或描述包含该关键词的服务卡片

#### Scenario: 按标签过滤
- **WHEN** 用户点击某个标签
- **THEN** 列表过滤为仅包含该标签的服务

### Requirement: 手动添加服务
系统 SHALL 提供弹窗表单让用户手动添加服务条目。

#### Scenario: 打开添加表单
- **WHEN** 用户点击"添加服务"按钮
- **THEN** 页面显示模态弹窗，包含 name、url、description、tags、icon 输入字段

#### Scenario: 提交有效表单
- **WHEN** 用户填写 name 和 url 后点击"保存"
- **THEN** 系统调用 POST `/api/services`，成功后关闭弹窗并刷新服务列表

#### Scenario: 表单验证
- **WHEN** 用户提交表单但 URL 格式不合法
- **THEN** 系统在字段旁显示错误提示，不提交请求

### Requirement: 编辑和删除服务
系统 SHALL 允许用户通过 UI 编辑和删除已注册的服务条目。

#### Scenario: 编辑服务条目
- **WHEN** 用户点击服务卡片上的"编辑"按钮
- **THEN** 页面显示预填充了当前数据的编辑表单；保存后更新条目

#### Scenario: 删除服务条目
- **WHEN** 用户点击"删除"并确认
- **THEN** 系统调用 DELETE `/api/services/:id`，成功后从列表中移除该卡片

### Requirement: 端口扫描 UI
系统 SHALL 在 UI 中提供端口扫描入口，展示扫描进度和结果。

#### Scenario: 触发扫描
- **WHEN** 用户点击"扫描本地端口"按钮
- **THEN** 页面显示扫描进度条，实时更新扫描进度百分比

#### Scenario: 展示扫描结果
- **WHEN** 扫描完成
- **THEN** 页面显示发现的开放端口列表，每条包含端口号、推测服务类型和"添加到注册表"按钮

#### Scenario: 一键添加扫描到的服务
- **WHEN** 用户点击某个扫描结果的"添加"按钮
- **THEN** 系统预填充表单并打开编辑弹窗，确认后加入注册表
